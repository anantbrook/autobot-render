DEPLOYMENT GUIDE

Deploy Your Bot to
Render in 6 Steps

Free 24/7 hosting for your automated Telegram channel bot — no server knowledge needed.

1
Push Your Bot to GitHub
Render deploys directly from GitHub. You need to put your bot files there first.
ℹ️
If you've never used GitHub: go to github.com, create a free account, then follow the steps below.
Terminal / Command Prompt
# 1. Install Git if you don't have it: https://git-scm.com # 2. Navigate to your bot folder cd path/to/autobot # 3. Initialize a git repo git init git add . git commit -m "Initial bot commit" # 4. Create a repo on GitHub.com (click + → New Repository) # Name it: auto-channel-bot | Set to Private ✓ # Copy the repo URL they give you, then: git remote add origin https://github.com/YOUR_USERNAME/auto-channel-bot.git git push -u origin main
⚠️
Make sure .gitignore is in your folder (it's included). This prevents your API keys from being uploaded. The config.py reads from environment variables — no real keys are in the code.

6
Auto-Deploy on Every Code Update
Whenever you push changes to GitHub, Render automatically redeploys your bot with zero downtime.
To update your bot at any time:
# Make your changes to any file git add . git commit -m "Updated posting schedule" git push # Render detects the push → auto-redeploys in ~2-3 min # Check live logs: Render Dashboard → your service → Logs
Render Dashboard → your service → Auto-Deploy should say Yes
🔍 How to Check Your Bot is Working
1. Render Logs: Dashboard → your service → Logs tab → see real-time output
2. Telegram Channel: Check posts are appearing on schedule
3. Service Status: Should show ● Live (not ● Failed)
🚑 Common Errors & Fixes
❌ TELEGRAM_BOT_TOKEN env var not set
→ Go to Render → Environment → make sure you added TELEGRAM_BOT_TOKEN
❌ Forbidden: bot is not a member of the channel
→ Add your bot as admin to the Telegram channel (Step 1c in SETUP_GUIDE)
❌ Build failed: no module named 'xyz'
→ Make sure requirements.txt is in your repo root and was pushed to GitHub
❌ Service keeps restarting
→ Check the Logs tab for the actual error. Most common: missing env var or bot not admin in channel
